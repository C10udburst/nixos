## Repository Structure & Configuration Flow

To navigate and manage this repository efficiently, understand the following file layout and configuration flow:

### 1. Main Entry Points
- **[flake.nix](flake.nix)**: The root flake definition. Declares external inputs (e.g., `nixpkgs`, `home-manager`, `plasma-manager`) and outputs. It defines host configurations such as `nixosConfigurations.cloudburst-desktop`, which imports the host's configuration module.
- **[apply](apply)**: A helper bash script that automates formatting Nix files (using `alejandra`), showing git diffs, running the `nixos-rebuild switch --flake .` command, and committing changes automatically under the new generation metadata when successful. Do not run this script blindly; it is indented for use by the human operator to ensure safe deployment of changes.

### 2. Hosts Configuration (`hosts/`)
Each machine has its own subdirectory in the `hosts/` folder:
- **[hosts/cloudburst-desktop/](hosts/cloudburst-desktop/)**:
  - **[configuration.nix](hosts/cloudburst-desktop/configuration.nix)**: Imports hardware configuration, `home-manager`, and the shared NixOS modules directory `../../modules/nixos`. Reads user-defined settings from `settings.nix` and provisions the NixOS system.
  - **[settings.nix](hosts/cloudburst-desktop/settings.nix)**: A single central source of truth attribute set containing global variables (e.g. `username`, `adminUsers`) and boolean toggles to enable or disable features/modules across NixOS and Home Manager.
  - **[home.nix](hosts/cloudburst-desktop/home.nix)**: The Home Manager configuration for the primary user. Imports the shared home modules directory `../../modules/home` and loads host-specific details.
  - **[hardware-configuration.nix](hosts/cloudburst-desktop/hardware-configuration.nix)**: The hardware-specific settings generated by NixOS.

### 3. Modular System & User Settings (`modules/`)
Configurations are divided into granular, reusable modules:
- **[modules/nixos/](modules/nixos/)**: NixOS-level system configurations (services, system packages, hardware configs).
  - **[default.nix](modules/nixos/default.nix)**: Imports all available modules in this directory, defines a `hostSettings` option, and maps options in `hostSettings` (from `settings.nix`) to set system settings defaults (e.g., enabling pipewire, tailscale, podman, etc.).
- **[modules/home/](modules/home/)**: Home Manager-level user-environment configurations (dotfiles, user-facing programs like shells, starship, git).
  - **[default.nix](modules/home/default.nix)**: Imports all user configuration modules, defines `hostSettings` and `homeSettings` options, and configures defaults based on values set in `settings.nix` (e.g., git, plasma, driftwm).
